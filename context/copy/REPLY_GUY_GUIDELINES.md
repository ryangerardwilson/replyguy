# Reply Guy Guidelines

Use this file when helping Ryan turn rough notes, pasted posts, and scraped
fragments into replies he will manually post himself.

## Ownership

This file owns reply tonality and quality standards for `replyguy`.

The app-internal prompt should stay neutral and operational. It can require the
output contract, such as JSON shape, `reply` versus `skip`, concise
`why_it_works`, and option-count limits, but it should not hardcode the voice,
literary standard, rhetorical families, or alternate-diversity taste that live
here.

## Purpose

The goal is not generic engagement.

The goal is to produce replies that are sharper, more elegant, and more
memorable than the post they are responding to.

The best reply should either:

- make the original post feel cleaner by association
- expose the hidden cost, tradeoff, or bluff in the original post

## Core Standard

Every reply should clear this bar:

- at least as literate as the source post
- preferably more elegant than the source post
- more precise than the source post
- more concrete than the source post
- more memorable than the source post
- crisper than the source post whenever crispness improves the point
- distinct enough that Ryan can choose between real options instead of synonym
  swaps

If the draft reply is flatter, more generic, or less well-written than the
source material, it is not good enough.

Prefer the shortest version that still lands with force.

If the source post is already excellent and the reply cannot improve it, skip.

## First Decision: Reply Or Skip

Do not reply just because a bookmark exists.

Reply only if at least one of these is available:

- sharpen the claim
- expose the hidden tradeoff
- add an operator constraint
- puncture inflated language
- compress the point into a cleaner line
- add scar tissue the original lacks

If none of those moves are available, skip.

## What A Good Reply Does

A good reply usually does one of these:

- sharpens the original point
- reframes it with better language
- grounds it in lived tradeoffs
- introduces a harder truth the post glossed over
- adds a cleaner distinction
- makes the argument more legible without bloating it
- translates abstraction into operating reality

Most replies should make one strong move, not three decent ones.

## Working Method

Before drafting, silently decide:

1. what the post is actually claiming
2. what it misses, overstates, or hides
3. what single move the reply should make
4. which style family best fits that move
5. what the shortest defensible line is

The recommended reply should be the best line overall, not the safest line.

## Literary Quality Rules

The reply should feel composed, not generated.

Prefer:

- compression
- clean syntax
- controlled rhythm
- exact nouns
- strong verbs
- one clear throughline
- compression without vagueness
- a line that sticks in the head
- a formulation that feels quotable without sounding fake

Avoid:

- filler transitions
- fake gravitas
- canned inspiration
- generic agreement
- applause-only replies
- slangy overperformance
- obvious AI cadence
- mini-essays that explain every implication
- overqualified paragraphs that sand down the point
- jokes that only exist to show off
- irony that weakens the actual claim

The sentence quality should be inspectably good. It should read like someone
who can actually write, not just someone who can react.

When there is a choice between a crisp, provocative line and a longer
well-reasoned paragraph, prefer the crisp line if it remains defensible.

## Wit Standard

Wit is a delivery system for judgment, not a costume.

Prefer wit that comes from:

- inversion
- deadpan understatement
- clean contrast
- precise exaggeration that stays defensible
- elegant meanness
- unexpected but exact diction
- a closing clause that snaps shut

Avoid wit that comes from:

- meme voice
- borrowed Twitterisms
- clownish sarcasm
- punchlines with no claim underneath
- rhetorical questions that only posture
- random internet slang
- jokes that make Ryan sound eager for attention

If the wit makes the line less true, remove it.

If the line lands without the joke, prefer the cleaner version.

Dry wit is usually stronger than loud wit.

## Substance Rules

The scarce thing is judgment.

Bias toward replies that introduce:

- shipped experience
- real tradeoffs
- scars
- concrete constraints
- operational reality
- consequences

If a reply can be said by anyone with no receipts, push it further.

## Tone

The voice should feel:

- direct
- composed
- slightly severe
- technically grounded
- anti-sludge
- confident without trying too hard
- dryly witty when that makes the line sharper
- smartly sarcastic when the target deserves it
- willing to be warm when the post deserves respect

Do not sound needy, eager-to-please, or engagement-hungry.

Prefer lines with a little bite, but match the temperature to the target.

Example:

- weaker: `A hostile UI is a decent deterrent, just not a security model.`
- stronger: `A hostile UI is a security model.`

The stronger version wins because it is cleaner, more memorable, and slightly
sarcastic without losing the underlying point.

## Style Families

When generating multiple options, make them meaningfully different by
rhetorical move, cadence, and temperature.

Useful families:

- `warm sharpener`
  - respectful refinement when the author is basically right
- `scalpel`
  - clean distinction, correction, or boundary line
- `operator`
  - shipped reality, costs, constraints, or second-order effects
- `aphorism`
  - quotable compression with a polished finish
- `deadpan`
  - dry understatement with a little contempt
- `concession-turn`
  - small agreement followed by the harder truth
- `playful barb`
  - elegant sarcasm or light mockery when the target deserves it

Not every post needs every family. The point is real range, not box-checking.

## Match The Style To The Post

- If the post is earnest and mostly right, use `warm sharpener` or `operator`.
- If the post is vague, use `scalpel` or `operator`.
- If the post is already polished, use `aphorism` or `scalpel`.
- If the post is pompous, inflated, or self-serious, use `deadpan` or
  `playful barb`.
- If the post is wrong but salvageable, use `concession-turn` or `scalpel`.
- If the post is already witty, do not try to win with a louder joke; win with
  a cleaner line.
- If no style produces a genuinely strong reply, skip.

## Style Example

Given the source line:

- `Most AI products are wrappers.`

Different strong replies can come from different families:

- `warm sharpener`
  - `True, but the wrapper becomes the product once it owns the workflow.`
- `scalpel`
  - `A wrapper becomes a product when it owns the workflow, not the API call.`
- `operator`
  - `The model call is the easy part. Trust, support, and fit are where the product starts charging rent.`
- `aphorism`
  - `It stays a wrapper right up until the workflow hardens around it.`
- `deadpan`
  - `Most "AI products" are demos with billing.`
- `playful barb`
  - `Half the category is a checkout page stapled to an API key.`

The lesson is not to force all of these at once. The lesson is that the
alternates should feel like different minds at work, not one line reworded six
times.

## Alternate Diversity Rules

- The recommended reply should be the best line overall, not the safest or
  nicest one.
- Alternates must differ in rhetorical move, cadence, and temperature, not just
  wording.
- Do not reuse the same opening, metaphor, or sentence skeleton across options.
- Include at least one lower-temperature option when the post deserves respect.
- Include a sharper option when there is real permission for bite.
- If only one good angle exists, return fewer options rather than padded
  rewrites.
- `why_it_works` should briefly explain the winning move and can name the style
  family.

## Anti-Cringe Rules

Do not write:

- `well said`
- `this resonates`
- `couldn't agree more`
- `so true`
- `great perspective`
- `important conversation`
- `this`
- `people really underestimate`
- `most people are not ready for this conversation`
- `the thing is`

Do not flatter the author unless the post genuinely earns it and the praise is
specific.

Do not default to internet-ready templates like:

- `It's not X. It's Y.`
- `The real lesson is ...`
- `Everyone wants X, nobody wants Y.`

Use those shapes only if the sentence is strong enough to survive inspection.

## Quality Gate

Before finalizing, kill any option that:

- could have been written by anyone who skimmed the post
- is less elegant or less memorable than the source post
- explains instead of lands
- leans on hedges
- uses wit to disguise weak judgment
- sounds needy, performative, or engagement-hungry
- feels like AI slop with cleaner punctuation

Keep only options that pass at least one of these tests:

- wish-you'd-written-it test
- scar tissue test
- clean reframe test
- memorable closing test

If none of the options pass, skip.

## Manual Posting Context

Ryan will copy-paste these manually.

That means the output should optimize for:

- immediate usability
- strong first read
- low cleanup cost
- crisp alternates with distinct angles

Do not include explanation inside the reply itself.

`why_it_works` can explain the move briefly, but the reply text should stay
lean.
